# homework-22

## Описание

Одностраничное приложение **(SPA) «Контакты»**, реализованное на **React + TypeScript**
с использованием **Redux + React Redux + Redux Thunk**.

Проект представляет собой рефакторинг приложения с локальным состоянием **(useState)**
в централизованное хранилище Redux с асинхронной загрузкой данных
и архитектурой **Feature-Sliced Design (FSD)**.

Проект построен с упором на:

- Redux как менеджер состояния
- Redux Thunk для асинхронной логики
- React Router v6
- Feature-Sliced Design (FSD)
- работу с селекторами и derived state
- аккуратную интеграцию Redux без переписывания UI

---

## Основной функционал

Контакты

- просмотр списка контактов

- фильтрация контактов:

- по имени (substring match)
- по группе

- страница контакта:

- фото
- номер телефона
- адрес
- дата рождения
- добавление и удаление контактов из избранного

Группы

- список групп

- страница группы:

- информация о группе
- список контактов, входящих в группу

Избранное

- список избранных контактов
- отображение в том же UI, что и основной список

Асинхронная загрузка данных

- данные загружаются через mock API
- используется Redux Thunk

для каждого запроса реализованы состояния:

- loading
- error
- loaded
- предусмотрена возможность повторной загрузки при ошибке

## Архитектура

Проект организован по принципам **Feature-Sliced Design (FSD)**:

- **app** — инициализация приложения, store, providers, роутинг
- **pages** — страницы приложения
- **components** — UI-компоненты (карточки, layout, меню)
- **entities** — доменные сущности (contacts, groups)
- **features** — бизнес-фичи (favorites, contacts-filter)
- **shared** — переиспользуемые утилиты, api, hooks
